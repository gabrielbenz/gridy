jQuery Gridy is a plugin that generates a highly customizable grid automatically.
Features:
home | scroll

JSON:
{"entityList": [{"id": 1, "name": "wbotelhos", "email": "gridy@wbotelhos.com"}], "total": 1}
- The JSON must have a list of object and an attribute with the total length.
Script:
$('#grid').gridy({ url: 'http://wbotelhos.com/gridy' });
- Just especify your URL that will returns the JSON;
- That's all, but you'll want to use other cool features.
HTML:
<div id="grid"></div>
- Where the grid will appears;
- In this case we're using the default template "gridy-default", but you can create your own. (:
Template:
<script id="template" type="text/x-jquery-tmpl">
   <div>
      <div>${name}</div>
      <div>${email}</div>

      <div class="button">
         <a href="#">like</a>
      </div>
    </div>
</script>
- The script that is a template type (text/x-jquery-tmpl) must have an ID to be hooked;
- The first div represents a row of the grid and will be repeated many times;
- Each div inside the wrapper will be considerated a col of the grid;
- If you are using some framework like Spring or EL (Expression Language), scape the '$' to avoid conflicts: \${ } or <c:out value="${ }"/>.
Query String:
key=__&page=__&sortname=__&sortorder=__&find=__&rows=__
- key: the term you want to search;
- page: the number of the page you want to view;
- sortname: the name of the column of the database you want to sort by;
- sortorder: the order you want to sort the result: ascending or descending;
- find: the name of the column you want to search;
- rows: the number of rows you want to display in each page;
- You can append more attributes using the 'params' option.
Server Side (Java):
@Get("/gridy")
public void gridy(String key, int page, String sortName, String sortOrder, String find, int rows) {

}
